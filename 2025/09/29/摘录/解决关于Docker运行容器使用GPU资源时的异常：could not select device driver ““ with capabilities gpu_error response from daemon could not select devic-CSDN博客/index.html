原因分析

上述异常问题可以大概推断出： Docker 容器无法选择正确的GPU设备驱动程序，可能是由于Docker运行时的配置或GPU设备驱动程序配置不正确所致。

大概在Docker19版本之后，Docker可以直接获得GPU资源的支持， 其支持不需要再单独安装 nvidia-docker ，它已被集成到Docker中。

3.推测原因

在 Docker容器 中使用宿主机的GPU资源，需要在执行 docker run 命令的时候添加 --gpus all 参数即可轻松实现容器使用宿主机的GPU资源。
