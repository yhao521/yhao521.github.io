用 iTerm2 的人很多，相关分享也很多，但介绍一些“藏得比较深”的功能分享很少见。iTerm2 最优秀的点就在于它打破了很多终端的传统的使用范式，增加了很多实用的、现代的、与图形界面相结合的功能，这些功能时常让人忽然意识到：“对哦，终端为什么不可以这样呢？”。 所以，本文分享一些其中个人觉得十分实用的技巧，也是 iTerm2 区别于绝大部分其它终端的独特功能。 Command Composer 输入长命令的时候，想修改中间的内容一点一点移动光标是不是十分痛苦？为什么不可以搞个文本编辑框呢？Composer 就是做这个的。点击 “View-Show Composer”或按 “Shift+Cmd+.”即可打开一个文本框，直接鼠标点击更改光标位置。 Snippets 很多复杂的常用命令需要自己搞个文档记录？iTerm2 也可以帮忙记录。在 iTerm 中按下 “Cmd+,” 打开 iTerm2 的设置，点击“Shortcuts”、“Snippets”，向其中添加常用的命令。 我一般喜欢从 Tool belt 中调用，按下 “Shift+Cmd+B” 可以打开 Tool belt，勾选 Snippets 让它展示出来，双击即可输入命令。 Triggers 大量的命令输出中如何快速找到一些特定的关键字或内容？Triggers 会按照自定义的配置从输出中捕获正则表达式，随后根据自定义的 Action 进行操作。我最常用的就是高亮 Error 和 Warning 的关键字。 打开设置： 2. 编辑规则，我的这些规则就是所有 Error 和 Exception 字眼的都红色高亮，所有 Warning 黄色高亮。 效果就是下图这个 “ModuleNotFoundError” 被标红了： 几个正则表达式： ^[A-Za-z\.]+(E|e)rror (W|w)arning [A-Za-z]+Exception 除了高亮，也可以将捕捉到的内容 capture output，然后可以做进一步处理，这个用法和特定任务有关，大家可以根据自身需要尝试。 tmux 集成 登录服务器的时候，如果希望（1）一个 session 分屏（2）掉线不让任务断掉，常规的方案就是使用 tmux 或 screen。但是 tmux 相当于在终端里套了个窗口管理器，还会占用一组快捷键前缀，需要花时间学习记忆（反正我是从来记不住）。但是既然 tmux 是在做窗口管理，为什么终端不可以做呢？iTerm2 可以做到窗口和 tmux 的窗口无缝衔接。 任意 ssh 登录一个服务器，输入以下命令即可进入 tmux。 $ tmux -CC 此时，我们使用常规 iTerm2 提供的分屏快捷键（1）左右分屏（Cmd+D）（2）上下分屏（Shift+Cmd+D），它在底层实际是做 tmux 分屏。如果这个时候服务器掉线，或我们直接关闭整个终端窗口假装掉线，再次登录到服务器时，可以输入以下命令重新回到刚刚的 tmux 窗口，进程仍然在。 $ tmux -CC attach Shell integration 这个功能需要额外安装，点击这里，然后一直 continue 即可。这个是安装到一个物理机器上的，可以是你自己的电脑，也可以是服务器。只要安装了，就都可以使用以下功能。 快速跳转到上一个 prompt 按下 “Shift+Cmd+上” 和 “Shift+Cmd+下” 即可快速在命令的 prompt 之间跳转。 终端内看图片 Shell integration 正常会在 /.iterm2/imgcat 放一个脚本，我们只需要在 ~/.bashrc 或 ~/.zshrc 中加入一个 alias： alias imgcat=$HOME/.iterm2/imgcat 然后就可以在终端内看图片： $ imgcat picture.png 服务器右键下载文件 服务器上也可以安装 Shell integration。然后在 ls 列出文件后，可以直接对一个文件“右键 - Download with scp”，使用 scp 下载文件。只不过很多时候需要自己配一下服务器的 hostname，稍微有些麻烦，所以只推荐在常用的固定服务器上使用这个功能。 在 自己的电脑 上添加服务器的 hostname： $ sudo bash -c ‘echo “ “ >> /etc/hosts’ AI 能力 iTerm2 3.5 版本开始增加了 AI 能力集成。需要使用我们自己的 API key，所以注意一下这个特性需要付费。 首先需要配置 LLM，这里以 通义千问 为例。 配置 1. 注册一下 Dashscope 获得 API key ，形如 “sk-123456789”。 2.打开 iTerm2 的设置 第三步，点击 Install，按照指引下载一个文件，这个文件是为了保证网络安全性。解压缩后放到 /Applications/ 目录下。 第四步，打开 AI 功能。 第五步，输入刚在 Dashscope 里获得的 API key。 第六步，根据需要更改 prompt。这里我用的比较简单：“Write a bash command for Unix that \(ai.prompt)”。 第七步，选择想使用的模型，比如 qwen-turbo qwen-max qwen2-72b-instruct codeqwen1.5-7b-chat 等，完整的列表见 这里 。不需要去点下拉菜单，直接把模型名字写进去就好。 第八步，设定 token 数，这个官网也有详细说明，如果填大了会返回报错信息告知你 token 应当是多少。 第九步，更改 API URL 为： dashscope.aliyuncs.com/ 一问一答 第一种方式为一问一答式的，比如一个命令不知道怎么输，问问 LLM。 打开第一节提到的 Composer，我们可以直接输入一个问题，点击 “Engage Artificial Intelligence” 即可回复。 交互式一步步指导 第二种方式为 codecierge，可以一步步指引。 按下 “Shift+Cmd+B” 打开 Tool belt，点击右上角三个点，选择 Codecierge。 输入一个想做的事情，例如“I want to modify the second last git commit message”。随后通义会告诉你先输入 git rebase -i HEAD2 ，然后在弹出的编辑器中做 reword，退出编辑器，会再次弹出编辑器让你修改 commit message。 这个功能的特点在于它会将你后来输入的命令及其输出自动发给 LLM，提高交互效率，不用你自己反复去网页上的 LLM 里复制粘贴。缺点是，要钱，不过其实价格很低了，一次问答也就几分钱。 Status bar 这个是个顺带一提的小功能。根据下图的指引找到 Status Bar 的配置。将上面的模块拖到下面： 即可实现在终端的底部（或顶部）显示一些小工具的功能： 设置顶部或是底部：
