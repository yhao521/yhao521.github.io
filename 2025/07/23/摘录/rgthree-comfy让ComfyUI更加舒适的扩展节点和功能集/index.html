rgthree-comfy:让ComfyUI更加舒适的扩展节点和功能集

rgthree-comfy是一个由开发者rgthree创建的ComfyUI扩展项目,旨在通过提供一系列实用的自定义节点和功能改进,让ComfyUI的使用体验变得更加舒适和高效。这个项目源于作者在使用ComfyUI过程中的个人需求,但其功能和优化也得到了广大用户的认可和欢迎。让我们一起来探索rgthree-comfy为ComfyUI带来的各种便利功能吧。

丰富多样的自定义节点

rgthree-comfy为ComfyUI添加了许多强大而实用的自定义节点,这些节点可以大大简化工作流程的构建过程。以下是其中一些代表性的节点:

Seed节点

Seed节点提供了一个直观的种子控制界面,其功能类似于Automatic1111的种子控制。用户可以通过设置种子值为”-1”来使用随机种子,或者设置任意其他数字作为固定种子。此外,节点还提供了快速随机化、使用上一次队列中的种子等快捷操作。

![[_resources/5f39bae91b842404e60e7ce34a850d56_MD5.png]]

Reroute节点

Reroute节点是对ComfyUI原有重路由节点的重大改进。它支持多个方向和尺寸的实际重路由,可以通过右键菜单来改变宽度、高度和连接布局,使工作流程更加整洁有序。

![[_resources/ac7efa9f3899102d48d9756a8a95edbe_MD5.png]]

Context/Context Big节点

Context节点系列可以在工作流程中传递和合并通用属性数据。与其他节点套件的”管道”相比,它提供了更简单的合并功能,并且通过在单个节点中同时组合和分解数据,更容易与标准节点互操作。

![[_resources/b1a73fba74f3721f8da8654b9e992645_MD5.png]]

Image Comparer节点

Image Comparer节点允许用户在同一个界面上比较两张图像。用户可以通过”滑动”或”点击”模式来切换查看两张图像的不同部分。

Power Lora Loader节点

这是一个超级简洁的Lora加载器节点,可以一次加载多个Lora模型,并快速切换每个模型的开启状态。用户可以通过”+ Add Lora”按钮添加任意数量的Lora,并通过右键菜单对每个Lora进行上下移动、开关切换或删除等操作。

Context Switch/Context Switch Big节点

这是一个强大的工作流分支节点,通过选择第一个非空/非null的Context输入来工作。它可以自动选择第一个非空的context继续执行,从而实现复杂的条件分支逻辑。

Fast Groups Muter节点

Fast Groups Muter是一个无输入节点,可以自动收集当前工作流中的所有组,并允许快速静音和取消静音组内的所有节点。用户可以通过节点属性来筛选、排序和配置显示的组。

实用的功能改进

除了丰富的自定义节点,rgthree-comfy还为ComfyUI带来了许多实用的功能改进和优化:

进度条

rgthree-comfy在应用窗口顶部添加了一个简洁的进度条,显示队列大小、当前提示执行进度以及多步骤节点的进度。用户可以通过rgthree-comfy设置来启用/禁用或调整进度条的高度。

ComfyUI递归优化

由于rgthree-comfy节点使构建更大、更复杂的工作流变得容易,作者对ComfyUI的递归执行进行了优化。这项优化可以显著减少复杂工作流的执行时间,为用户节省大量计算资源。

“队列选定输出节点”右键菜单选项

有时用户可能只想执行工作流中的一两个特定输出节点路径,而不是整个工作流。rgthree-comfy添加了一个右键菜单选项,允许用户选择并只执行特定的输出节点。

长组合框中的自动嵌套子目录

rgthree-comfy可以自动检测长组合列表(如Load Checkpoint)中的顶级子目录,并将其分解为子目录,使选择更加方便。

组标题中的快速静音/旁路切换

在组标题的右上角添加了静音和/或旁路切换图标,方便用户快速切换当前正在查看的组的状态。

导入单个节点小部件(拖放)

允许用户通过拖放先前生成的图像/JSON工作流来覆盖相同节点的小部件(匹配相同ID和类型)。这对于在使用相同通用工作流的多个生成之间导入特定数据(如种子或提示)非常有用。

高级技巧:利用静音实现复杂工作流

rgthree-comfy的许多强大功能来源于对”静音”机制的巧妙运用。静音是正确实现使用Context Switch节点的多路径工作流的基础。与其他扩展提供的开关不同,rgthree-comfy的Context Switch通过选择第一个非空context来传递,并正确地使用静音来使先前的节点为空,从而在设置正确时不会造成额外的工作。

理解静音的关键在于理解图的流程。当处理工作流时,执行实际上是从每个”输出节点”(如Preview Image、Save Image等)开始向后移动的。因此,要停止某个路径的执行,我们需要静音该路径的输出节点,而不是开头的节点。

![[_resources/a7a6194ef28e19a58a6ef4495e5cef50_MD5.png]]

上图展示了一个使用Context、Context Switch和Fast Muter节点的强大组合示例。通过Context Switch节点,我们可以按优先顺序输入context。Fast Muter节点则充当仪表板,允许快速启用和禁用连接的节点。这种组合允许我们构建复杂的条件工作流,例如生成一系列基本预览,然后选择其中一个进行放大并保存到磁盘。

安装和配置

之后启动ComfyUI即可。

rgthree-comfy提供了一些配置选项,允许用户根据需要启用或禁用某些功能。您可以通过右键单击图表的空白部分并选择 rgthree-comfy > Settings (rgthree-comfy) 来访问设置,或者在ComfyUI设置对话框中点击 rgthree-comfy settings 。

结语

rgthree-comfy为ComfyUI带来了众多实用的自定义节点和功能改进,极大地提升了用户的工作效率和使用体验。无论是通过强大的Context节点系列简化数据流,还是利用Image Comparer节点方便地比较图像,亦或是使用进度条和递归优化来提高性能,rgthree-comfy都体现了作者对提升ComfyUI使用体验的不懈追求。

对于那些希望构建更复杂、更高效的ComfyUI工作流的用户来说,rgthree-comfy无疑是一个值得尝试的扩展。它不仅能够简化工作流程,还能激发用户探索ComfyUI更多可能性的热情。随着项目的不断发展和社区的贡献,我们可以期待rgthree-comfy在未来为ComfyUI带来更多令人兴奋的功能和改进。
