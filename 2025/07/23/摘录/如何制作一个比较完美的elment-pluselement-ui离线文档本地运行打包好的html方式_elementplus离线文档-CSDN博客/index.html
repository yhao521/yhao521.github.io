本文介绍了一种Element Plus组件库的离线部署方法。主要包括从GitHub下载代码、解压缩并使用VSCode配合LiveServer插件预览，或通过安装http-server实现本地服务器访问。

摘要生成于 C知道 ，由 DeepSeek-R1 满血版支持， 前往体验 >

官方给的网站有时候不好用，自己制作的方法

第一步先去下面的网站下载代码，注意一定看看有没有zh-CN这个 文件夹 ，

GitHub - element-plus/element-plus at gh-pages

第二步，把下载好的代码解压到一个自己想要的文件夹

第三步1，使用vscode+live server插件，打开解压好的文件夹，注意主要是这里的文件夹，最好是下一级就包含index.html这些官方打包好的文件的文件夹。打开zh-CN这个文件夹，点击里面的index.html文件，使用open with live server即可

![[_resources/26e43889588cc316fac138586c578ab4_MD5.png]] ![[_resources/a9aa8fb38063cdbb6153d7da4eb2100c_MD5.png]]

第三步2，嫌vscode打开麻烦的可以试试这种方法，参考下面的教程

本地的html文件怎么运行在本地服务器上_普通html如何在本地开启一个服务运行-CSDN博客

安装 http-server ，最好全局安装
https://www.npmjs.com/package/http-server

之后在需要运行的html项目文件夹中，

![[_resources/8d46fd44eb3b0f6c11c26c3ea2ed6076_MD5.png]]

打开cmd，输入http-server，http-server会提供一个8080端口，打开 http://127.0.0.1:8080 就是html项目，

这里可以创建run_element- plus.bat文件，来一键实现打开cmd输入http-server命令的操作

start http-server -p 55555 —–> 修改端口的地址，只要在下图的范围之内就没问题

![[_resources/e8b5c619bc276d14e518148637895827_MD5.png]]

start http://127.0.0.1:55555 —–> 打开浏览器

这里的 http://127.0.0.1 可能可以修改成localhost这种，具体教程可以参考.

http-server - npm

这样就能直接一键实现elment-plus的离线了

![[_resources/958d51b0ff84b39d8fc2f383083085ce_MD5.png]]

element -ui的使用也是一样的原理，下载源码，在index.html同目录下创建一个启动的bat文件，内容和上面的一样，使用时点击一下运行即可

https://github.com/ElemeFE/element/tree/gh-pages
